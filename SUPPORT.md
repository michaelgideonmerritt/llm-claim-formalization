# Support

## Community Support Model

This project is provided as open-source software under the MIT License. While community contributions are welcome, please understand the support expectations:

### What You Can Expect

- **Bug Reports**: We welcome bug reports via GitHub Issues. Please include:
  - Clear description of the issue
  - Steps to reproduce
  - Expected vs actual behavior
  - Python version and environment details
  - Minimal code example demonstrating the issue

- **Documentation**: The README and code examples are the primary support resources. Please consult these first.

- **Feature Requests**: You may submit feature requests via GitHub Issues. However, please note:
  - Feature requests will be reviewed but may not receive immediate response
  - Implementation timelines are not guaranteed
  - The project maintainer has limited availability

- **Pull Requests**: Community contributions via pull requests are appreciated:
  - Please include tests for new functionality
  - Follow existing code style (no comments in code, self-documenting)
  - Expect variable review times

### What You Should Not Expect

- **Guaranteed Response Time**: Issue responses may take days or weeks depending on maintainer availability
- **Feature Development on Request**: Feature requests do not guarantee implementation
- **1-on-1 Support**: The maintainer does not have capacity for individual support requests
- **Commercial Support**: No paid support options are available at this time

### Self-Service Resources

Before opening an issue, please check:

1. **README.md** - Installation, usage, and architecture documentation
2. **examples/quickstart.py** - Working code examples
3. **Existing GitHub Issues** - Your question may already be answered
4. **Code Tests** - `tests/` directory shows usage patterns

### How to Help Yourself

This library is designed to be self-contained and well-documented:

- All core functionality is in `src/llm_claim_formalization/core/extraction.py`
- Backend implementations are in `src/llm_claim_formalization/backends/`
- Demo server code is in `src/llm_claim_formalization/demo/server.py`
- Code is written to be self-documenting (minimal comments, clear naming)

Reading the source code is often the fastest way to understand behavior.

### Contributing

If you find this library useful and want to contribute:

- **Documentation improvements** are always welcome
- **Bug fixes** with tests are highly valued
- **Example additions** help other users
- **Performance optimizations** with benchmarks are appreciated

### Philosophy

This project follows the principle of "batteries included, support optional":

- The library works out of the box with minimal configuration
- Documentation provides clear usage examples
- Community support is best-effort, not guaranteed
- Users are encouraged to read and understand the code

### Contact

For non-support inquiries (collaboration, research, etc.), you may open a GitHub discussion.

Please do not email the maintainer directly for support questions - use GitHub Issues instead.
